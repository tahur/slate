# Code of Conduct

Slate is committed to a respectful, inclusive, and professional contributor community.

## 1. Expected Behavior

1. Be respectful in all discussions and reviews.
2. Focus criticism on code and behavior, not people.
3. Assume good intent, ask clarifying questions, and communicate clearly.
4. Welcome contributors with different backgrounds and experience levels.
5. Keep feedback specific, actionable, and professional.

## 2. Unacceptable Behavior

1. Harassment, hate speech, or discriminatory language.
2. Personal attacks, threats, or intimidation.
3. Deliberate disruption of discussions, reviews, or project operations.
4. Publishing private information without consent.
5. Repeated bad-faith behavior after moderator guidance.

## 3. Scope

This Code of Conduct applies to:
1. GitHub issues, pull requests, and discussions
2. project chat/community spaces
3. any public or private communication representing Slate

## 4. Enforcement

Maintainers are responsible for enforcing this policy.

Possible actions:
1. warning
2. temporary suspension
3. permanent removal from project spaces

## 5. Reporting

To report unacceptable behavior, contact maintainers through project channels and include:
1. what happened
2. where it happened
3. links/screenshots if available

Reports will be reviewed confidentially where possible.

## 6. Maintainer Standards

Maintainers must:
1. apply rules consistently
2. document enforcement decisions when needed
3. avoid retaliatory behavior

This document may evolve as the community grows.
